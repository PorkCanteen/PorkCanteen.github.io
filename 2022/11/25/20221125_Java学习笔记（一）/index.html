前言

作为前端开发者，了解一些后端语言知识，在工作日常的前后端交互过程中能够提升一定的效率，也能让自己对整个软件开发的流程及原理有更加深入的了解，同时也能拓宽自己的知识面。
本人从基础开始，利用空闲时间学习一些Java的基础知识。

CMD

CMD-在windows中，利用命令行的方式操作计算机

CMD常用命令

  • 盘符名称 + 冒号 – 切换到某盘
    E:
  • dir – 查看当前路径下的内容
    dir
  • cd 目录 – 进入单极目录
    cd java
  • cd .. – 回退到上一级目录
    cd..
  • cd 目录1\目录2... – 进入多级目录
    cd java\detail
  • cd \ –回到盘符根目录
    cd \
  • cls –清屏
    cls
  • exit – 退出命令提示符窗口
    exit

数据类型

  • 分为基本数据类型和引用数据类似
  • 基本数据类型
    1. 整数
      a. byte：-128到127
      b. short：-32768到32767
      c. int（默认）：10位数
      d. long：19位数（在数据值的后面需要加一个L作为后缀）
    2. 浮点数
      a. float：-3.401298e-38到3.402823e+38（在数据值的后面需要加一个F作为后缀）
      b. double（默认）：-4.9e-324到1.797693e+308
    3. 字符
      a. char：0-65535
    4. 布尔
      a. boolean：true false

运算符

算术运算符

+ - * / %
在代码中，若有小数参与计算，结果可能不精确

  1. 数字相加
  • 数字运算时，若数据类型不一样不能运算，需要转为一样的才能运算
  • double > float > long > int > short > byte
  • 隐式转换(自动类型提升)：取值范围小 -> 大

    取值范围小和取值范围大的进行运算，小的先提升为大的，再进行运算
    byte short char 三种类型的数据在运算时，都会直接先提升为 int 再计算

  • 强制转换()：取值范围大 -> 小

    若把取值范围大的数值赋值给范围小的变量是不允许的，需要强制转换
    目标数据类型 变量名 = (目标数据类型) 被强制转换的数据;

  1. 字符串相加
  • 当 + 操作中出现字符串，则这个 + 就是字符串连接符，而不是算数运算符，它会将前后数据进行拼接产生一个新的字符串
  • 连续进行操作时，从左到右逐个进行
  1. 字符相加
  • byte short char 三种类型的数据在运算的时候，都会直接先提升为 int 再进行计算
  • 当 “字符 + 字符” 和 “字符 + 数字” 时，会把字符通过 ASCII 码表查询到对应的数字再进行计算

Java内存分配

  • 栈
    方法运行时使用的内存，比如 main 方法运行，进入方法栈中执行
  • 堆
    存储对象或者数组，new来创建的都存储在堆内存
  • 方法区
    存储可以运行的class文件
  • 本地方法栈
    JVM在使用操作系统功能的时候使用，与开发无关
  • 寄存器
    给CPU使用，与开发无关

方法

  • 方法(method)是程序中最小的执行单元
  • 重复的代码、具有独立功能的代码可以直接抽取到方法中
  • 可以提高代码的复用性及可维护性

方法的注意事项

  • 方法间是平级关系，不能互相嵌套定义
  • 方法的编写顺序与执行顺序无关
  • 方法的返回值类型为void，没有返回值的方法可以省略return不写，或只写return

方法的重载

在同一个类中定义了多个同名方法，这些同名方法具有同种功能。每个方法具有不同的参数类型或参数个数，这些同名方法就构成了重载关系。
